# NetXMS Weekly Digest — 13 to 19 April 2026

Summary of changes merged into the main branch and issue tracker activity
between Monday 13 April and Sunday 19 April 2026.

## At a Glance

- **32 commits** landed during the period, by 5 authors
  (Victor Kirhenshtein, Alex Kirhenshtein, Tatjana Dubrovica, Filipp Sudanov,
  and external contributor dnpowers).
- **14 issues opened** (9 enhancements, 1 bug, 4 discussion/other).
- **20 issues closed** — including long-standing feature requests and
  several newly-reported bugs fixed the same week.
- Version bumped from **6.1.1** to **6.2** (`1fc3f11`, `c21ede9`).

## Highlights

### New capabilities

- **LLDP via agent** — a new `lldpd` subagent exposes `lldpd` neighbour data
  to the server, which now consumes LLDP information coming from agents in
  addition to SNMP (`5e809a2`, `c726ee0`, closes #3178).
- **SNMP MIB enum display** — DCI values can now be rendered using the
  textual enum labels from the SNMP MIB instead of raw integers
  (`60e2bc0`, closes #3183).
- **apk package retrieval** — Linux subagent now supports listing installed
  packages on Alpine / apk-based distributions (`d805ac4`, closes #3176).
- **Object tools on more object classes** — object tools can now be attached
  to interfaces, sensors, and access points; SSH command tools are allowed
  on interfaces (`f53daa4`, `7482831`, #2631, #2217).
- **NXSL: scheduled polls** — scripts can initiate polls directly
  (`143573b`, closes #2686).
- **NXSL: read-only security context** — scripts that should not mutate
  state now run under a restrictive context (`cbced8e`, closes #3191).
- **Windows event processing** — log parser callbacks accept named
  variables; the server-side Windows-event receiver extracts variables
  from the received XML and propagates them into the generated event
  (`da00e3c`, #2978). Handling of the `<Binary>` data tag was also added
  (`f5a3f3d`).
- **Template policies on clusters** — applying a template to a cluster now
  propagates the policies to its member nodes (`c3af91e`, #2560).

### Bug fixes

- `DataSeries.getCurrentValue` regression from `8072ac8a63` fixed, with
  unit tests covering `min` / `max` / `avg` / `invert`
  (`9141624`, `85502b3`, fixes #3189).
- 60 Hz CPU-burning repaint loop in `ViewFolder` / `ViewStack` caused by
  `SWT.WRAP` on `topRight` toolbars (`4abe6e3`, fixes #3175).
- Rack / chassis drawing bug caused by a missing lazy load in
  `ImageProvider.renderImage` (`7cdd2a9`).
- Chassis elements now show tooltips in rack view
  (`5a90282`, fixes #2471).
- Log-parser position persistence bug on Windows agent
  (`76b1431`, fixes #3126).
- Percentage column in bar-chart dashboard element can be switched off
  (`2f586f8`, fixes #3036).
- Multiple compatibility fixes for newer Asterisk versions
  (`03368b6`, fixes #3190).
- Server-side bugs in the Windows event processor (`e1de55a`).
- `NullPointerException` in `LogParserRuleEditor` when adding a rule
  with the `POLICY` parser type (`859aa25`).
- Typo in the `pgsql.ddr` driver reference corrected by external
  contributor (`5ed2f15`).

### Infrastructure, UI, tooling

- **OpenAPI / WebAPI** — `operationId` added to all 143 endpoints; info
  description added; all 144 vacuum lint errors eliminated
  (`925b706`). Duplicate-description bug on *AI Chat* / *AI Saved
  Prompts*, missing *AI Management* tag declaration, and 35 untagged
  endpoints all fixed (`3f7373a`).
- **V5 data-table migration** improved: shorter delays between migration
  cycles and cleaner debug messages (`24fed12`).
- **Log viewer** — new `LCF_SYNC_OBJECTS` column flag for interface
  synchronisation; connection-history-log *Port* / *Old Port* columns
  now resolve interface names correctly when interface sync is disabled
  (`f14140a`).
- **Web API** — IP address included in the object-summary payload
  (`f86d932`).
- **Windows installer** picks up the Matrix driver (`c20a8a4`).
- Prometheus subagent escapes special characters in metric argument
  values (`57b2fe3`).
- Updated logo (`2e48ae9`) and `publiccode.yml` (`9eed7aa`, version
  6.1.0, GPL-2.0-or-later, adds de/es languages, spec bumped to 0.5).
